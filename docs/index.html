09 · Interview Preparation
Likely Interview Questions
Master these answers before your technical round.
What is agentic AI and how is it different from a simple LLM call?▼
Agentic AI refers to systems where an LLM autonomously takes multi-step actions — calling tools, making decisions, looping on results — rather than producing a single response. Unlike a one-shot LLM call, an agent perceives state, plans a sequence, executes tools, and adapts based on outputs. In this project, the Planner Agent breaks a user query into steps, the Data Agent executes pandas operations, and the system iterates until a complete answer is assembled.
How do your agents communicate with each other?▼
Agents communicate via structured message passing using LangGraph's state graph. Each agent reads from a shared state dictionary and writes its outputs back. The Planner writes a task plan; the Data Agent reads it, executes analysis, and writes results; the Insight Agent reads those results to generate text. This avoids tight coupling — any agent can be swapped without changing the others.
Why multi-agent instead of one large LLM prompt?▼
Single large prompts hit context limits, are difficult to debug, and mix concerns. Multi-agent architectures enable specialization (each agent is an expert in one task), parallelism (agents can run concurrently), observability (you can log each agent's exact input/output), and modularity (swap or upgrade one agent independently).
How did you handle data preprocessing challenges?▼
The Data Agent runs a preprocessing pipeline: (1) schema validation on upload, (2) null imputation using median for numeric and mode for categorical, (3) IQR-based outlier detection with flagging rather than deletion, (4) date parsing with timezone normalization, (5) type casting and column renaming to a standard schema.
How does your system scale to large datasets or many users?▼
FastAPI uses async endpoints so I/O-bound operations don't block the event loop. For large datasets, the Data Agent uses chunked Pandas processing and can dispatch to Dask. Docker + cloud deployment enables horizontal scaling. Redis caches repeated analysis results. FAISS memory queries are O(log n) even at millions of vectors.
What is the reflection loop and why does it matter?▼
The reflection loop is a self-critique mechanism where an agent evaluates its own output before passing it downstream. The Insight Agent generates a finding, then runs a second prompt asking "Is this conclusion supported by the data?" If the critique fails, the agent regenerates — dramatically reducing hallucinated or unsupported insights.
